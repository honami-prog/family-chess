## 1. アプリ概要

**Family Chess — Wooden Traveler Series** は、家族・友人グループ専用のクローズドなチェス＆将棋プラットフォームです。  
1つのアプリでチェス・将棋の両方を、**オンライン対局・AI対戦・タクティクス練習・棋譜解析・グループチャット**まで一括して楽しめます。

### 主な特長

| 特長 | 内容 |
|------|------|
| 🌐 **オンライン対局** | Firebase Realtime Database による即時同期。複数の卓を同時進行可能 |
| 🤖 **AI対戦** | Stockfish 18（チェス）・Fairy-Stockfish（将棋）によるレベル10段階 |
| 🎯 **タクティクス** | Lichess API連携で無限に出題（チェス）／オリジナル問題集（将棋） |
| 📊 **棋譜解析** | エンジンによる全手評価・精度スコア・評価値グラフ |
| 💬 **チャット** | DM・グループチャット、日英自動翻訳機能付き |
| 👨‍👩‍👧 **キッズモード** | 子供向けのヒント表示・シンプルなUI |
| 🇯🇵🇺🇸 **日英バイリンガル** | UI全体を日本語・英語で切り替え可能 |
| 🪵 **木製ボードデザイン** | Wooden Traveler Series コンセプトの統一デザイン |

## 3. プレイヤー管理

### 3-1. 入室パスワード

- アプリ起動時にパスワード（`family2025`）を入力して入室
- 一度入室するとブラウザの `localStorage` に認証情報を保存（再訪問時スキップ）

### 3-2. プレイヤー選択

- 登録済みメンバー一覧からプロフィールを選択してプレイ開始
- デフォルトメンバー：Thomas / Clark / Honami / おじいちゃん / キッズ（5名）

### 3-3. プロフィール設定

各プレイヤーが以下を個別に設定できます：

| 設定項目 | 内容 |
|----------|------|
| **表示名** | 20文字以内で変更可能 |
| **表示言語** | 🇯🇵 日本語 / 🇺🇸 English の切り替え |
| **アバター** | プリセット絵文字・動物・キャラクター画像から選択、または URL 指定 |
| **キッズモード** | オン/オフで子供向けUIに切替 |

### 3-4. メンバー管理

- 管理者はメンバーの追加・削除・キッズフラグ設定が可能
- メンバーデータは `localStorage` に保存

## 4. チェス機能

### 4-1. オンライン対局（マルチプレイヤー）

- **3卓同時** のゲームを Firebase で管理（No.1 / No.2 / No.3）
- 同じゲームに複数のプレイヤーが参加・観戦可能
- 手番ごとに Firebase がリアルタイム同期 → 相手の指し手が即座に反映

#### 対局開始フロー
1. 卓を選択 → 対局開始モーダル
2. 白番・黒番・観戦のいずれかを選択
3. 相手が入室したら対局スタート

#### ゲーム機能
| 機能 | 内容 |
|------|------|
| **合法手表示** | 選択した駒の移動可能マスを緑/ドットで表示 |
| **キャスリング** | 適切な条件でキングサイド・クイーンサイドキャスリング対応 |
| **アンパッサン** | 通過ポーンの特殊取り対応 |
| **ポーン昇格** | 8段目到達時に Q/R/B/N から選択するダイアログ表示 |
| **チェック判定** | チェック時にアナウンス表示＋サウンド、チャット通知 |
| **チェックメイト/ステールメイト** | 自動判定・勝敗モーダル表示 |
| **投了** | いつでも投了可能 |
| **1手戻す** | 相手に取り消し申請（相手の承認制） |
| **再戦リクエスト** | 対局終了後に再戦ボタン |
| **取得駒表示** | 双方が取った駒を盤の上下に表示 |
| **指し手履歴** | 代数記法で全手を記録・表示 |
| **前の手プレビュー** | 手順表の手をクリックで局面を確認 |
| **サウンド** | 移動・取り・チェック・勝利の4種類 |

### 4-2. 対面モード（Face to Face）

- **1台の端末を2人で使う**対面モード
- 盤を縦に並べ、相手側の駒を180°回転表示
- 自分の手番のみ操作可能（相手の盤面は操作不可）
- 各プレイヤーに**投了ボタン**を個別配置

### 4-3. AI対戦

- **Stockfish 18** エンジン（WebWorker経由）
- **10段階の難易度**：

| レベル | 日本語 | 英語 | Skill Level |
|--------|--------|------|-------------|
| 1 | キッズ | Kids | 0 |
| 2 | 入門 | Beginner | 2 |
| 3 | 初級 | Easy | 4 |
| 4 | 初中級 | Easy+ | 6 |
| 5 | 中級 | Medium | 9 |
| 6 | 中上級 | Medium+ | 11 |
| 7 | 上級 | Hard | 14 |
| 8 | 高段 | Hard+ | 16 |
| 9 | マスター | Expert | 18 |
| 10 | エキスパート | Master | 20 |

- AI の色（白/黒）を選択可能
- 盤面の向きは AI 側が上に自動フリップ
- 対局終了後に棋譜解析へ移行可能
- 勝利時：トロフィーモーダル 🏆 / 敗北時：ロボットモーダル 🤖

### 4-4. フリー練習モード

- AI なしで盤面を自由に動かして練習
- 1手戻し・盤面リセット・定石再生・タクティクス学習と連携

### 4-5. 盤面操作

| 機能 | 内容 |
|------|------|
| **盤面フリップ** | 黒番でプレイするとき等、盤面の向きを手動変更 |
| **フルスクリーン** | ボタン1つでゲーム画面を全画面表示（スマートフォン最適化） |
| **PC/モバイル自動切替** | 画面幅に応じてレイアウト自動変更 |
| **PC 2カラムレイアウト** | 盤面左・情報/チャット右のデスクトップ最適レイアウト |

### 4-6. キッズモード（チェス）

- 相手に取られそうな駒に**赤枠**で警告表示
- ポーン昇格は**Q/R のみ**表示（選択肢を簡略化）
- 右クリックで駒の説明ポップアップ
- 子供向けのアナウンス文面

### 4-7. 盤面デザイン

- 木目調ボード（Wooden Traveler Series テーマ）
- 角のビス飾り・装飾フレーム
- 優雅な駒の影エフェクト
- アンバー/クリーム色のマス目配色

## 5. 将棋機能

### 5-1. オンライン対局（マルチプレイヤー）

チェスと同様に Firebase Realtime Database でリアルタイム同期。

#### 将棋特有の機能
| 機能 | 内容 |
|------|------|
| **持ち駒システム** | 取った駒を再利用（打ち駒）対応 |
| **成り判定** | 敵陣でのプロモーション選択ダイアログ |
| **成り強制** | 退路のない位置での強制成り |
| **打ち歩詰め禁止** | 歩を打っての詰みを禁止する特殊ルール |
| **二歩禁止** | 同じ列に2枚の歩を打てない |
| **合法手計算** | 将棋固有の全禁じ手を考慮した合法手計算 |
| **持ち駒表示** | 盤の上下に持ち駒を表示（数量付き） |
| **駒の回転表示** | 先手は正立、後手は180°回転（対面モード） |

### 5-2. 対面モード（将棋）

- 1台の端末で2人対面プレイ
- 先手・後手の駒方向を自動反転表示
- 持ち駒エリアも上下分割

### 5-3. AI対戦（将棋）

- **Fairy-Stockfish WASM**（将棋対応エンジン）
- 10段階難易度（チェスと同じスケール）
- 先手/後手の選択可能
- USI プロトコルでエンジンと通信

### 5-4. キッズモード（将棋）

- 相手に取られそうな駒に赤枠警告
- 成り選択を「なる/ならない」のシンプルなダイアログ
- 子供向け文面

## 6. タクティクス（チェス）

### 6-1. 概要

**Lichess Puzzle API** と連携し、世界最大の無料チェス教材データベースからリアルタイムで問題を取得します。  
タクティクスは「練習モード」内のボタンから起動します。

### 6-2. 問題取得・重複排除

- Lichess API `GET /api/puzzle/next` から問題を動的取得
- 解いた問題IDを **localStorage** に最大200件保存（セッションをまたいで重複排除）
- 同一問題が出た場合は最大6回まで自動再取得
- 次の問題を**バックグラウンドでプリフェッチ**（正解後の次の問題遷移を即時化）

### 6-3. フィルター機能

#### 難易度フィルター
| 設定 | Lichess API パラメータ |
|------|----------------------|
| すべて | なし |
| Easy | `difficulty=easiest` |
| Normal | `difficulty=normal` |
| Hard | `difficulty=hardest` |

#### テーマフィルター（13種類）
| 日本語 | 英語 | API パラメータ |
|--------|------|---------------|
| すべて | All | なし |
| 1手詰め | Mate in 1 | `themes=mateIn1` |
| 2手詰め | Mate in 2 | `themes=mateIn2` |
| 3手詰め | Mate in 3 | `themes=mateIn3` |
| フォーク・両取り | Fork | `themes=fork` |
| ピン | Pin | `themes=pin` |
| サクリファイス | Sacrifice | `themes=sacrifice` |
| スキュアー | Skewer | `themes=skewer` |
| 陰の攻撃 | Discovered Attack | `themes=discoveredAttack` |
| ダブルチェック | Double Check | `themes=doubleCheck` |
| そらし | Deflection | `themes=deflection` |
| 誘い込み | Decoy | `themes=decoy` |
| エンドゲーム | Endgame | `themes=endgame` |

- 難易度 × テーマを**組み合わせて指定**可能（例：Easy × サクリファイス）

### 6-4. 正誤判定

- ユーザーが駒を動かすと即座に正解/不正解を判定
- **正解（✅）**：駒が正解位置に移動、正解モーダル表示
- **不正解（❌）**：不正解モーダル表示

### 6-5. 正解・不正解モーダル

| 正解時 | 不正解時 |
|--------|----------|
| ✅ と「正解！/ Correct!」 | ❌ と「不正解 / Incorrect」 |
| 問題レーティング（★○○○） | 「もう一度 / Try Again」ボタン |
| 「▶ 次の問題」ボタン | 「答えを見る / Show Answer」ボタン |
| スプリングポップアニメーション | 「▶ 次の問題」ボタン |

### 6-6. ヒント・答え表示

| 機能 | 内容 |
|------|------|
| **ヒント 💡** | 解答手の移動先マスを**緑色**でハイライト |
| **答えを見る** | 解答手の移動元・移動先マスを**青色**でハイライト |
| **盤面の向き** | 黒番の問題は盤面を自動フリップ（黒が下に） |

### 6-7. ナビゲーション

| ボタン | 動作 |
|--------|------|
| ⏭ スキップ | 現在の問題をスキップして次へ（解答を記録しない） |
| ◀ 前の問題 | 1つ前の問題に戻る |
| もう一度 / Retry | 同じ問題を最初から再挑戦 |
| ✕ 終了 | タクティクスモード終了、通常練習モードへ |

### 6-8. 成績記録（Firebase）

- プレイヤーごとに `tactics/{playerName}/{puzzleId}` に以下を記録：
  - 問題ID、難易度、正解/不正解、ヒント使用有無、試行回数、解答日時

### 6-9. ヘッダー表示

問題のヘッダーに以下を表示：
- 難易度バッジ（Easy=緑 / Normal=オレンジ / Hard=赤）
- テーマバッジ（選択中の場合、青グレー）
- レーティング（★○○○）
- 手番説明（「白番です / White to move」 or 「黒番です / Black to move」）

## 9. 棋譜解析

### 9-1. 概要

対局後の棋譜を **Stockfish**（チェス）または **Fairy-Stockfish**（将棋）で全手分析します。

### 9-2. 解析手順

1. 対局終了後に「解析」ボタンをクリック
2. エンジンが各手の評価値を計算
3. 結果を Firebase に保存（デバイス間共有）

### 9-3. 解析結果の表示

#### 評価値グラフ
- 横軸：手順番号、縦軸：評価値（センチポーン）
- 白優勢（上）・黒優勢（下）を視覚化
- 手番の境界線を表示
- クリックした手番の局面を盤面プレビューで確認

#### 手ごとの評価
各手に以下のラベルを付与：

| ラベル | 説明 |
|--------|------|
| ✨ Brilliant | 例外的に優れた手 |
| ✅ Best | 最善手 |
| 👍 Good | 良い手 |
| 😐 Inaccuracy | 少し惜しい手（CPL 30未満） |
| ⚠️ Mistake | 悪手（CPL 100未満） |
| ❌ Blunder | 大悪手（CPL 100以上） |

#### 精度スコア（Accuracy）
- 白・黒それぞれの平均精度を 0〜100% で表示
- Lichess と同様の計算式（Win Percentage ベース）
- 精度に応じたコメント（例：「驚異的な精度です！」）

#### 局面プレビュー
- 解析リストの各手をクリックすると、その局面の盤面を表示
- 白・黒の取得駒も表示

### 9-4. 解析履歴

- 過去の解析をリスト表示（`AnalysisList` コンポーネント）
- Firebase から取得、日時・対局者名・勝者を表示
- 過去の解析を再閲覧可能

### 9-5. バッジ・通知

- 未読の解析結果がある場合、「解析」タブにバッジ（未読数）を表示
- デバイス間でバッジ状態を Firebase 同期

## 10. チャット

### 10-1. DM チャット

- プレイヤー同士の1対1ダイレクトメッセージ
- Firebase Realtime Database にメッセージを保存
- 1ルームあたり最新300件を保持（古いメッセージは自動削除）

### 10-2. グループチャット

- 複数メンバーのグループルーム作成
- ルーム名の設定可能
- メンバー2人以上から作成可能

### 10-3. 自動翻訳機能

- 日本語メッセージ → 英語に自動翻訳
- 英語メッセージ → 日本語に自動翻訳
- **Lingva Translate API** 使用
- 翻訳結果を Firebase にキャッシュ（同じテキストの再翻訳を防止）
- 表示言語が英語のプレイヤーには日本語メッセージの英訳を表示（逆も同様）

### 10-4. メッセージ通知

- 未読メッセージがある場合、「チャット」タブにバッジを表示
- 未読数をデバイス間で Firebase 同期

### 10-5. チャットUI

- メッセージに送信者名・送信時刻（日本/ニューヨーク両タイムゾーン）を表示
- 相手メッセージと自分のメッセージを左右で区別
- キッズモードの送信者には 🐥 アイコンを表示
- 「なんでもかいてね！」（キッズ向けプレースホルダー）

### 10-6. ゲーム内チャット

- 対局中のチェック通知はチャットルームにも自動送信
- ゲームパネル内にミニチャット表示（最新メッセージ）

### 11-1. デザインシステム

**Wooden Traveler Series** として統一されたデザイン言語：

| **カード** | 角丸16〜24px、ボックスシャドウ付き、アイボリー背景 |

### 11-2. レスポンシブ対応

| 画面幅 | レイアウト |
|--------|------------|
| `> 1024px` | PC 2カラム（盤面左 + 情報/チャット右） |
| `≤ 1024px` | モバイル 1カラム縦積み |

### 11-3. フルスクリーンモード

- ボタン1つで盤面を全画面表示
- ステータスバー・ナビゲーションを非表示にして盤面最大化
- モバイルで特に有効（`position: fixed; inset: 0`）
- フルスクリーン内にすべての操作ボタンを内包

### 11-4. 2タイムゾーン時計

- 画面上部に常時表示
- **日本時間**（JAPAN）と**ニューヨーク時間**（NEW YORK）を同時表示
- 海外メンバーとのプレイ時に時差を意識しやすい設計

### 11-5. ナビゲーション

サイドバー（PCは左固定、モバイルは上部タブ）：

| タブ | 内容 |
|------|------|
| ♟ チェス | チェス対局・AI・タクティクス |
| 🀄 将棋 | 将棋対局・AI・タクティクス |
| 📝 練習 | 定石ガイド・タクティクスパターン |
| 📊 解析 | 棋譜解析・解析履歴 |
| 💬 チャット | DM・グループチャット |
| ⚙️ 設定 | プロフィール・メンバー管理 |

### 11-6. キッズモード UI 差異

| 機能 | キッズモード |
|------|-------------|
| 取られそうな駒 | 赤枠で警告表示 |
| 駒説明 | 右クリック（長押し）でポップアップ |
| 文面 | ひらがな中心・短文 |
| チャット | 「なんでもかいてね！」プレースホルダー |
| ポーン昇格 | Q/R の2択のみ（B/N を非表示） |
| 対局者バッジ | キッズには 🐥 マーク付き |

### 11-7. バッジ・アバターシステム

- **キングバッジ**：チェスで白番・黒番を使った場合のバッジ画像（`/badges/king-white.webp`, `king-black.webp`）
- **将棋バッジ**：将棋で先手・後手を使った場合のバッジ（`/badges/shogi-black.webp`, `shogi-white.webp`）
- タップ/クリックでバッジの拡大プレビュー
- アバター画像も同様に拡大プレビュー対応

### 11-8. モーダル・アニメーション

| モーダル | アニメーション |
|----------|---------------|
| 勝利モーダル | フェードイン |
| タクティクス正解/不正解モーダル | スプリングポップ（`cubic-bezier(0.34, 1.56, 0.64, 1)`） |
| ポーン昇格選択 | スライドイン |
| アバターピッカー | オーバーレイフェード |

### 11-9. サウンドエフェクト

| イベント | サウンド |
|----------|----------|
| 駒の移動 | コン（木の音） |
| 駒を取る | 取り音 |
| チェック | アラート音 |
| 勝利・投了 | 勝利音 |

- 遅延ロード（初期ロードのパフォーマンスに影響させない）
- ユーザー操作時に `Web Audio API` で再生

## 12. セキュリティ・データ管理

### 12-1. アクセス制御

- アプリ入室にはパスワード認証が必要
- Firebase のデータは家族グループ内で共有（一般公開なし）

### 12-2. データ保存先

| データ | 保存先 |
|--------|--------|
| チェス/将棋対局データ | Firebase Realtime Database |
| チャットメッセージ | Firebase Realtime Database |
| 棋譜解析結果 | Firebase Realtime Database |
| タクティクス成績 | Firebase Realtime Database |
| 翻訳キャッシュ | Firebase Realtime Database |
| プレイヤー設定 | localStorage（ブラウザローカル） |
| 盤面フリップ状態 | localStorage |
| タクティクス既出問題ID | localStorage（最大200件） |

### 12-3. データ自動削除

- チャットルームは最新**300件**を超えた古いメッセージを自動削除
